Product Requirements Document (PRD) Product Name: FindLink Objective Build a platform to connect missing and found person reports and assist identification. Problem Statement No centralized system, low visibility, and difficulty matching missing with found persons. Solution Overview Users post missing/found reports; system suggests matches; verified notifications are sent. Target Users Families, public helpers, NGOs, admins. Core Features Authentication, User Profiles, Missing/Found Reports, Matching System, Notifications, Map, Search, Admin Panel, Communication, Privacy. Authentication Signup with email/phone, OTP verification, login, password reset, roles (User/Admin). User Profile Profile info, report management, privacy controls. Missing Reporting Name, age, gender, location, date, description, photos, case ID, status. Found Reporting Photo, location, date, notes, anonymous option. Matching System Face + text comparison, match score, suggested matches, human verification. Notifications Email/SMS alerts for matches and updates. Map &amp; Location Display cases on map, nearby search. Search &amp; Filters Search by name/location, filter by age, gender, status. Admin Panel Moderate reports, verify matches, manage users. Communication Comments, reporting abuse, optional messaging. Privacy &amp; Safety Hide contact info, consent, data deletion, blocking. Non-Functional Fast performance, secure (HTTPS, hashing), mobile-friendly. Tech Stack Frontend: React, Backend: Node/Django, DB: MongoDB/PostgreSQL. Database Users, MissingReports, FoundReports, Matches. User Flow Signup ΓåÆ Post ΓåÆ Match ΓåÆ Verify ΓåÆ Notify ΓåÆ Resolve. Metrics Cases, matches, resolution rate, engagement. Risks False matches, fake reports, privacy issues. Future Mobile app, AI improvements, integrations. MVP Basic auth, posting, search, simple matching.
